# Plugin replication via manifest

Three small scripts (`claude-snapshot`, `claude-replicate`, `claude-promote`) implement declarative cross-machine alignment of `claude plugin marketplace`/`claude plugin install` state — without relying on Syncthing for the `~/.claude/` directory.

## Mental model

| File | Role |
|---|---|
| `shared.json` | **Source of truth.** Marketplaces + plugins that MUST be on every machine. Curated, intentional. |
| `snapshots/<host>.json` | **Per-machine state.** Generated by `claude-snapshot`. Reveals hostname + locally-installed items. **Gitignored by default** (privacy — see `snapshots/.gitignore`). |

## Workflows

### First-time setup (per fork)

```bash
cp claude/manifest/shared.json.example claude/manifest/shared.json
$EDITOR claude/manifest/shared.json     # add the marketplaces + plugins you want everywhere
bash install.sh                          # deploys the 3 scripts to ~/.local/bin/
```

### On each machine — see what's installed

```bash
claude-snapshot                          # writes claude/manifest/snapshots/$(hostname).json
```

### See what's missing vs the source of truth

```bash
claude-replicate                         # diff vs shared.json (read-only)
claude-replicate --apply                 # actually install missing items
```

### Promote a locally-installed item to the source of truth

```bash
claude-promote some-plugin@some-marketplace   # auto-edits shared.json + commits suggested
```

## Why this beats syncing `~/.claude/` via Syncthing

- `~/.claude/` is multi-GB (per-project state, plugin caches, telemetry, sessions, file-history) — most of which is per-machine, regenerable, or arch-specific. Syncing the whole tree is mostly wasted bandwidth.
- The **portion that matters** (skills, agents, settings, manifest) is already under `dotfiles/claude/` (this repo) and ships via git + `install.sh`.
- The **part that didn't have transport** before — the list of enabled plugins/marketplaces — is what these scripts handle declaratively, with an explicit curation step.

## Privacy note on snapshots

`claude/manifest/snapshots/<host>.json` reveals (a) your machine's hostname, (b) the full set of plugins that machine has enabled. The default `.gitignore` blocks committing them — keep them local. Override only if your fork is **private** and you want machine-by-machine reproducibility in git history; in that case set `ALLOW_SNAPSHOT_COMMIT=1` and remove the gitignore rule.
